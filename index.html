  • O QUE É O GIT?

    É um sistema de controle de versões distribuído, utilizado tanto na fase de desenvolver um software, quanto para registrar o histórico de edições dos arquivos em geral


  • PARA QUE SERVE O GIT?

    Com o git, é possível desenvolver projetos em que pessoas colobaram simultaneamente, editando e criando arquivos sem que haja a sobrescrita das alterações feitas


  • COMO SURGIU?

    Esse projeto de código aberto foi desenvolvido no ano de 2005, por Linus Torvalds, o grande criador do kernel do sistema operacional Linux. Ele foi projetado visando a segurança e flexibilidade

  • PRINCIPAIS COMANDOS

    Git init: cria um novo repositório do Git. Git add: inclui as alterações feitas que vão entrar no próximo commit

    Git commit: salva as alterações


  • OUTROS COMANDOS

    Git branch: para criar, listar ou excluir branches, que são ramificações que permitem o trabalho paralelo num mesmo projeto. As branches são muito importantes no mundo git por permitir essa colaboração aberta e conjunta


  • PULL E PUSH

    A diferença consiste em que o git push serve para empurrar os commits feitos no computador para o remoto, enquanto o git pull puxa as alterações feitas no remoto para o computador local


GITHUB



O GitHub é uma ferramenta muito popular na área de TI, acomodando mais de 25 milhões de usuários. Ele é um serviço baseado em nuvem que hospeda um sistema de controle de versão chamado Git, permitindo que aconteçam inúmeras colaborações e alterações em projetos compartilhados, enquanto mantém um registro detalhado do seu progresso. O slogan do GitHub é "Social Code Host", ou seja, "Hospedeiro Social de Códigos" - e isso implica dizer que ele armazena códigos de programação, produzidos por diferentes programadores e os compartilham como se fosse uma rede social. Assim, qualquer usuário da plataforma pode divulgar seus trabalhos e os outros membros da comunidade GitHub podem fazer contribuições.
